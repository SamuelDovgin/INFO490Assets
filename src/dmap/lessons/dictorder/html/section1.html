Dictionary Order

As you now know, the Python dictionary is just an unordered collection of key/value pairs. Since dictionaries are considered to be unordered collections, there's no need to build a dictionary in sorted order. In general there is no way to guarantee that the first item you add to a dictionary is the first item returned. This lesson is all about sorting the items in dictionary.

Coder's Log : A dictionary by definition is unordered. However in Python version 3.6 dictionaries will maintain insertion order. But you should not rely on this behavior (your code could run by someone using 3.5).

Sorting

Since dictionaries have both keys and values AND since the values can be any valid Python data type, sorting is not as simple as sorting a list.

We will step you through some of the common ways. It's not important to memorize the syntax; but you should know what you need to do to build a sorting process that reaches deep inside your dictionary.

Simple List Sorting

With a list you could just use the sorted function which works on lists (in ascending order -- the default):

values = [200, 100, 300]
print(sorted(values))
